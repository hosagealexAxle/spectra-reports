SPECTRA Research Platform

Project Status — March 5, 2026 (v4.3 — CEIA Pricing Intelligence Report Type)

Executive Summary

SPECTRA (Strategic Physical Ecosystem: Comprehensive Threat-detection Research Agent) is an AI-powered research platform for the weapons detection industry. It runs as a macOS Electron desktop app with a Node.js + Prisma backend, Claude (Anthropic) as the primary LLM, and a React dashboard accessible both locally and via Tailscale Funnel (public HTTPS) for remote testers.

The system generates deep, unbiased strategic research reports using Claude Opus 4.6. Two report types available: Full Market Report (40 API calls, 11 sections, ~191 web searches) and CEIA Pricing Intelligence Report (18 API calls, 10 sections, ~158 web searches with anti-bot spacing). Both include checkpoint/resume for fault tolerance, Deep Fetch web intelligence for current data, and a persistent chat interface with Claude Sonnet 4.6 and a knowledge base (Key Facts) for contextual memory.

New (Feb 26–27): Tailscale Funnel remote access with server-side password authentication. SHA-256 hashed password with crypto.timingSafeEqual(). Cookie-based sessions (30-day HttpOnly cookie). Auth middleware runs before static file serving — blocks dashboard, API, and all assets for unauthenticated remote users. Localhost/Electron requests bypass auth. Detects Funnel-proxied requests via X-Forwarded-For header. Dashboard API_BASE fixed to use window.location.origin for Funnel compatibility (was hardcoding port 4100). First external beta tester connected successfully.
Critical Fix (Feb 25): Replaced raw fetch() with @anthropic-ai/sdk for all Claude API calls. Node.js native fetch (undici) has a known connection pool bug that kills consecutive large API requests after 4–5 calls (SDK issue #712, Node.js #55330). The SDK handles connection management correctly and includes patches for this exact failure mode. Also adopted pre-fetch architecture: all Tavily searches run upfront before any Claude calls, completely separating HTTP workloads. Backend runs on system Node (/opt/homebrew/bin/node) instead of Electron’s bundled Node for fetch reliability.

The report engine generates deep, unbiased strategic research reports using Claude Opus 4.6. Each report makes 40 Claude API calls across 5 phases. Web intelligence is gathered in a 3-phase Deep Fetch pre-fetch that runs ~191 Tavily searches + full page extraction from vendor websites before any Claude calls begin.

Web Search Integration (Deep Fetch v4)

  • 3-phase pre-fetch architecture: ALL Tavily searches + page extractions run upfront before Claude calls, completely separating HTTP workloads
  • ~191 Tavily credits per report: ~57 static searches + ~85 per-company searches + ~14 extract credits + ~8 deep advanced searches + ~15 discovered company searches
  • includeRawContent: "markdown" on all searches — gets full page text at zero extra credit cost (free Tavily feature)
  • fetchPages() via Tavily Extract API — pulls full content from vendor websites (1 credit per 5 URLs)
  • discoverNewCompanies() — parses A1a output to find companies not in seed registry, generates targeted queries for each
  • Results formatted as == DEEP WEB INTELLIGENCE == blocks (rawContent preferred) or == EXTRACTED COMPANY PAGES == for page extractions
  • Non-fatal: if Tavily fails on any step, that step proceeds with the original prompt
  • Cache: 30-minute TTL prevents duplicate searches
  • Credit tracking: logged to ActionLog at end of each report (web_search_credits)
  • Cost: ~$1.91/report (191 credits × $0.01). Free tier covers ~5 reports/month (1,000 credits).

Key Design Decisions

  • Zero vendor bias: No vendor names appear anywhere in research prompts. Claude discovers and evaluates all vendors independently. Each vendor spotlight gets equal web searches.
  • Live web intelligence: Tavily API grounds every research step in current data (2025–2026), eliminating stale training data reliance.
  • Date-anchored: Every prompt includes "As of February 2026" to ensure current data focus.
  • Retry logic: 5 retries with 20s exponential backoff via Anthropic SDK. Client is reset on connection errors to force fresh connections.
  • Context passing: Phase 2 receives relevant Phase 1 outputs (4K–12K chars focused summaries). VS steps receive 6K A1aContext + per-company deep-fetched data.
  • Rate limiting: Max 1 report per 2 rolling days, 5 per calendar month. Admin override available.
  • Estimated cost: ~$10 per report (Claude Opus) + ~$1.91 (Tavily Deep Fetch) = ~$12 total.
  • Generation time: ~150 minutes (6 min pre-fetch + ~145 min Claude calls).

Known Gotchas

  • Backend is ESM ("type": "module") — use dynamic import() not require(). backend-runner.js is CJS bridge.
  • Express 5 + new path-to-regexp: NO app.get("*") wildcards. Use app.use() middleware for SPA fallback.
  • NEVER use raw fetch() for Claude API calls. Node.js undici has a known bug killing connections after 4–5 consecutive large requests. Always use @anthropic-ai/sdk.
  • Backend runs on system Node (/opt/homebrew/bin/node), not Electron’s bundled Node. Electron’s ELECTRON_RUN_AS_NODE has unreliable fetch for sustained API workloads. backend-runner.js is copied to app.getPath("userData") at launch since system Node can’t read from asar.
  • npm link / file: symlinks break inside Electron asar. Before each Electron build, run rm -rf node_modules/@hosage/web-search && cp -R ~/Desktop/shared-utils node_modules/@hosage/web-search to replace symlink with real copy.
  • Pre-fetch web searches before Claude calls. Never interleave Tavily and Claude HTTP requests — run all searches first, cache results, then run Claude calls clean.
  • Electron loads from file:// — window.location.hostname is empty. API_BASE checks for empty string.
  • Dashboard dist path: dev=../../dashboard/dist, packaged=process.resourcesPath/dashboard/dist. Server checks both.
  • .env is gitignored. Packaged app reads from ~/Library/Application Support/Spectra/.env.
  • Auto-migration in backend-runner.js via execFileSync of prisma migrate deploy before server start.
  • Dashboard uses Vite 7. Rebuild dashboard BEFORE rebuilding Electron (dashboard/dist gets bundled).
  • Report generation takes ~150 minutes (6 min pre-fetch + ~145 min Claude calls). Use caffeinate -d -i -s & to prevent Mac sleep during generation.
  • Report rate limit: 1 per 2 days, 5 per month. Admin override: user=admin, password hash stored in ADMIN_OVERRIDE_HASH env var (no plaintext in source).
  • Tavily free tier: 1,000 credits/month. At ~191/report (Deep Fetch) + chat usage, budget ~5 reports before paid tier.
  • Step counter goes +2 over TOTAL_STEPS during generation (A1a_1 and A1a_2 increment in Phase 2 pre-fetch, then skip-as-cached in Phase 1). Final DB record correctly shows 40. Cosmetic only.
  • After changing PWA meta tags or manifest, user must delete and re-add PWA on iPhone home screen.
  • Prisma client must be regenerated (npx prisma generate) if node_modules are copied from another project.
  • CORS is restricted to explicit origins: localhost:4100, 127.0.0.1:4100, Tailscale IP (100.84.51.101:4100), and Funnel domain. Update whitelist in server.js when adding new access points.
  • Tailscale Funnel proxies through localhost. Auth middleware must check X-Forwarded-For header to distinguish real local (Electron) requests from Funnel-proxied remote requests. Without this check, Funnel bypasses the password gate.
  • Auth middleware must run BEFORE express.static. Otherwise static dashboard files (HTML/JS/CSS) are served without authentication and only API calls are blocked.
  • Dashboard API_BASE must use window.location.origin for remote access. Hardcoding port 4100 fails through Funnel (which serves on port 443).
  • Funnel requires Terminal window to stay open (unless started with --bg flag). Mac must stay awake (use caffeinate). Spectra.app must be running.
  • Start Funnel: /Applications/Tailscale.app/Contents/MacOS/Tailscale funnel 4100. Stop: Ctrl+C or funnel --bg off.
  • GATE_HASH is now required. No hardcoded fallback. If missing from .env, server logs a warning and rejects all remote login attempts. Set in both backend/.env and ~/Library/Application Support/Spectra/.env.
  • ADMIN_OVERRIDE_HASH replaces plaintext ReportMe26. Pre-hash with SHA-256 and store hex in .env. Uses crypto.timingSafeEqual() for constant-time comparison.
  • Login rate limited: 5 attempts per minute per IP via express-rate-limit. Returns 429 Too Many Requests after threshold.
  • @hosage/web-search must be a real copy, not a symlink. Before every Electron build: cd backend && rm -rf node_modules/@hosage/web-search && cp -R ~/Desktop/shared-utils node_modules/@hosage/web-search. Symlinks break inside asar archives.
