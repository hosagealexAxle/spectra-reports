SPECTRA Research Platform

Project Status — February 25, 2026 (v3)

Executive Summary

SPECTRA (Strategic Physical Ecosystem: Comprehensive Threat-detection Research Agent) is an AI-powered research platform for the weapons detection industry. It runs as a macOS Electron desktop app with a Node.js + Prisma backend, Claude (Anthropic) as the primary LLM, and a React dashboard accessible both locally and remotely via Tailscale on iPhone.

The system generates deep, unbiased 11-section strategic research reports using Claude Opus 4.6, with 40 API calls per report across 5 phases for maximum depth. It includes checkpoint/resume for fault tolerance, Tavily web search for current data, and a persistent chat interface with Claude Sonnet 4.6 and a knowledge base (Key Facts) for contextual memory.

SUCCESS (Feb 25, 4:06 PM): Full 40-step research report generated successfully. 1.7MB HTML output with 488 section headings, live web intelligence from 54 Tavily searches, and comprehensive vendor/regulatory/field coverage. Cost: ~$4 — an 85% reduction from the original $28–30 estimate. Achieved through reduced max_tokens (16384 → 10000), prompt splitting, and the Anthropic SDK. At $4/report, all 5 monthly reports cost $20 total vs. $28 for a single report under the old engine. Checkpoint/resume system proved itself mid-run — auto-resumed from step 17 after an Anthropic 500 error with zero lost progress or wasted tokens.
New (Feb 25): Tavily web search integrated via shared @hosage/web-search npm package. 21 of 40 report steps receive live web intelligence (54 Tavily searches per report). Chat endpoints in both Spectra and Axle detect current-events questions and inject real-time web results. Checkpoint/resume system saves progress after every step — failed reports resume from last checkpoint automatically.
Critical Fix (Feb 25): Replaced raw fetch() with @anthropic-ai/sdk for all Claude API calls. Node.js native fetch (undici) has a known connection pool bug that kills consecutive large API requests after 4–5 calls (SDK issue #712, Node.js #55330). The SDK handles connection management correctly and includes patches for this exact failure mode. Also adopted pre-fetch architecture: all 54 Tavily searches run upfront before any Claude calls, completely separating HTTP workloads. Backend runs on system Node (/opt/homebrew/bin/node) instead of Electron’s bundled Node for fetch reliability.

Web Search Integration

  • Pre-fetch architecture: ALL 54 Tavily searches run upfront before any Claude calls, completely separating HTTP workloads
  • 54 searches across 21 of 35 steps (research/data steps only). 14 synthesis steps receive no searches.
  • Results cached in memory, formatted as == CURRENT WEB INTELLIGENCE == blocks, prepended to each Claude prompt
  • Non-fatal: if Tavily fails on any step, that step proceeds with the original prompt
  • Cache: 30-minute TTL prevents duplicate searches
  • Credit tracking: logged to ActionLog at end of each report (web_search_credits)
  • Cost: ~$0.43/report (54 × $0.008). Free tier covers ~18 reports/month (1,000 credits)

Key Design Decisions

  • Zero vendor bias: No vendor names appear anywhere in research prompts. Claude discovers and evaluates all vendors independently. Each vendor spotlight gets equal web searches.
  • Live web intelligence: Tavily API grounds every research step in current data (2025–2026), eliminating stale training data reliance.
  • Date-anchored: Every prompt includes "As of February 2026" to ensure current data focus.
  • Retry logic: 5 retries with 20s exponential backoff via Anthropic SDK. Client is reset on connection errors to force fresh connections.
  • Context passing: Phase 2 receives relevant Phase 1 outputs (4K–12K chars focused summaries).
  • Rate limiting: Max 1 report per 2 rolling days, 5 per calendar month. Admin override available.
  • Estimated cost: ~$28–30 per report (Claude Opus) + ~$0.43 (Tavily) = ~$28–31 total.
  • Generation time: ~45–90 minutes (logged automatically).

Chat Web Search

The Spectra chat endpoint detects current-events questions using keyword matching (current, recent, latest, 2025, 2026, news, etc.) and automatically runs a Tavily web search, injecting results into the system prompt before the LLM call.

  • Max results: 3 per query
  • Topic: news | Time range: past week
  • Max context: 3,000 chars injected into system prompt
  • Non-fatal: chat proceeds normally if search fails

Known Gotchas

  • Backend is ESM ("type": "module") — use dynamic import() not require(). backend-runner.js is CJS bridge.
  • Express 5 + new path-to-regexp: NO app.get("*") wildcards. Use app.use() middleware for SPA fallback.
  • NEVER use raw fetch() for Claude API calls. Node.js undici has a known bug killing connections after 4–5 consecutive large requests. Always use @anthropic-ai/sdk.
  • Backend runs on system Node (/opt/homebrew/bin/node), not Electron’s bundled Node. Electron’s ELECTRON_RUN_AS_NODE has unreliable fetch for sustained API workloads. backend-runner.js is copied to app.getPath("userData") at launch since system Node can’t read from asar.
  • npm link / file: symlinks break inside Electron asar. Before each Electron build, run rm -rf node_modules/@hosage/web-search && cp -R ~/Desktop/shared-utils node_modules/@hosage/web-search to replace symlink with real copy.
  • Pre-fetch web searches before Claude calls. Never interleave Tavily and Claude HTTP requests — run all searches first, cache results, then run Claude calls clean.
  • Electron loads from file:// — window.location.hostname is empty. API_BASE checks for empty string.
  • Dashboard dist path: dev=../../dashboard/dist, packaged=process.resourcesPath/dashboard/dist. Server checks both.
  • .env is gitignored. Packaged app reads from ~/Library/Application Support/Spectra/.env.
  • Auto-migration in backend-runner.js via execFileSync of prisma migrate deploy before server start.
  • Dashboard uses Vite 7. Rebuild dashboard BEFORE rebuilding Electron (dashboard/dist gets bundled).
  • Report generation takes 45–90 minutes. Use caffeinate -d -i -s & to prevent Mac sleep during generation.
  • Report rate limit: 1 per 2 days, 5 per month. Admin override available.
  • Tavily free tier: 1,000 credits/month. At 54/report + chat usage, budget ~18 reports before paid tier.
  • After changing PWA meta tags or manifest, user must delete and re-add PWA on iPhone home screen.
  • Prisma client must be regenerated (npx prisma generate) if node_modules are copied from another project.
  • CORS is open (origin: "*") — Tailscale provides network-level security.
